TS学习-接口

在 TypeScript 中，接口（interface）用于定义对象的结构和行为。接口不仅可以为类定义约定，还可以约束函数、数组等多种数据结构。通过接口，可以确保对象符合某种结构，使代码更加类型安全和可维护。

接口

面向对象语言中，接口（Interfaces）是一个很重要的概念，它是对行为的抽象，而具体如何行动需要由类（classes）去实现（implement）。

接口定义

接口通常用于定义对象的结构。通过接口，可以指定对象的属性及其类型。

接口属性

可选属性

在定义一些结构时，一些结构的某些字段的要求是可选的，有这个字段就做处理，没有就忽略，所以针对这种情况，TypeScript提供了可选属性，通过在属性名后添加 ? 标记为可选属性。

只读属性

接口中的属性可以使用 readonly 关键字声明为只读。只读属性在对象创建后不能被修改。

属性索引

接口可以定义可索引类型（Indexable Types），用来表示具有索引的对象，如数组或对象字典。索引签名的语法为 [index: type]: type。

函数类型接口

对象的方法

对象的方法共有三种写法。

函数

interface 也可以用来声明独立的函数。

构造函数

interface 内部可以使用new关键字，表示构造函数。

类实现接口

实现（implements）是面向对象中的一个重要概念。一般来讲，一个类只能继承自另一个类，有时候不同类之间可以有一些共有的特性，这时候就可以把特性提取成接口（interfaces），用 implements 关键字来实现。这个特性大大提高了面向对象的灵活性。

接口继承

接口继承接口

接口可以使用extends关键字继承其他接口，从而实现接口的扩展。可以通过继承将多个接口组合在一起形成新的接口。

接口继承类

interface 还可以继承 class，即继承该类的所有成员。当接口继承一个类时，它会继承该类的属性和方法，但是不会继承类的具体实现。这种继承方式常用于定义一个接口，使其兼容特定类及其子类的结构。

接口继承类型

接口继承类型主要用于将类型别名的特性合并到接口中。这种方式在 TypeScript 4.1 及更高版本中被支持。在接口继承类型别名时，接口会继承类型别名的属性结构，但不能继承联合类型或原始类型的直接定义。

接口继承交叉类型

接口可以继承交叉类型，来组合多个类型别名的结构。

接口不能继承联合类型

接口拓展

接口可以扩展来创建复杂的结构。使用交叉类型也可以实现类似的效果。

接口合并

在 TypeScript 中，可以对同一个接口进行多次声明，TypeScript 会自动将它们合并为一个接口。

接口泛型

接口可以使用泛型，使其更具通用性。泛型接口允许定义一个适用于多种类型的接口
