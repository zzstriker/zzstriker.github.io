TS学习-数据类型

TypeScript 继承了 JavaScript 的类型，在这个基础上，定义了一套自己的类型系统。

数据类型

在语法层面，缺省类型注解的 TypeScript 与 JavaScript 完全一致。因此，可以把 TypeScript 代码的编写看作是为 JavaScript 代码添加类型注解。

在 TypeScript 语法中，类型的标注主要通过类型后置语法来实现：“变量: 类型”

基本数据类型

JavaScript 中将值分成 8 种类型，包括：number、boolean、string、null、undefined、symbol、bigInt、object。TypeScript 继承了 JavaScript 的类型设计，以上 8 种类型可以看作 TypeScript 的基本类型。

number类型

TypeScript 中的 number 类型涵盖整数和浮点数，与 JavaScript 一致。同时，TypeScript 还支持十进制、十六进制、二进制、和八进制等不同数值表示方法。

boolean类型

boolean类型只包含true和false两个布尔值。

string类型

string 表示文本数据。TypeScript 支持双引号 (")、单引号 (')、模板字符串（用反引号 `` 标识）。模板字符串允许嵌入表达式和多行字符串。

undefined 类型和null 类型

undefined 和 null 是两种独立类型，它们各自都只有一个值。

undefined 类型只包含一个值undefined，表示未定义（即还未给出定义，以后可能会有定义）。null 类型也只包含一个值null，表示为空（即此处没有值）。

如果没有声明类型的变量，被赋值为undefined或null，它们的类型会被推断为any

undefined和null既是值，又是类型。作为值，它们有一个特殊的地方：任何其他类型的变量都可以赋值为undefined或null。

bigInt类型

bigInt是ES6中新引入的数据类型，它是一种内置对象，它提供了一种方法来表示大于 $ 2^{53} -1 $ 的整数，BigInt可以表示任意大的整数。使用 BigInt 可以安全地存储和操作大整数，即使这个数已经超出了JavaScript构造函数 Number 能够表示的安全整数范围。

可以看到，最终返回了true，这就是超过精读范围造成的问题，而BigInt正是解决这类问题而生的

这里需要用 BigInt(number) 把 Number 转化为 BigInt，同时如果类型是 BigInt ，那么数字后面需要加 n。

在TypeScript中，number 类型虽然和 BigInt 都表示数字，但是实际上两者类型是完全不同的

symbol类型

symbol 是 ES6 新增的一种基本数据类型，它用来表示独一无二的值，可以通过 Symbol 构造函数生成。

在TypeScript中使用symbol就是指定一个值的类型为symbol类型

TypeScript 中还有一个 unique symbol 类型，它是symbol的子类型，这种类型的值只能由Symbol()或Symbol.for()创建，或者通过指定类型来指定变量是这种类型。这种类型的值只能用于常量的定义和用于属性名。需要注意，定义 unique symbol 类型的值，必须用 const 而不能用let来声明。

复杂数据类型

array类型

TypeScript 数组有一个根本特征：所有成员的类型必须相同，但是成员数量是不确定的，可以是无限数量的成员，也可以是零成员。在 TypeScript 中有两种定义数组的方式：

  1. 直接定义：通过 number[] 的形式来指定这个类型元素均为number类型的数组类型，推荐使用这种写法。
  2. 数组泛型：通过 Array<number> 的形式来定义

如果数组成员的类型比较复杂，可以写在圆括号里面

JavaScript 规定，const命令声明的数组变量是可以改变成员的，很多时候确实有声明为只读数组的需求，即不允许变动数组成员。TypeScript 允许声明只读数组，方法是在数组类型前面加上readonly关键字。

对象

在 TypeScript 中，对象是一种包含多个属性和方法的结构。

在 TypeScript 中，可以通过 {} 定义一个简单的对象类型。对象类型的每个属性都可以指定类型

对象的属性可以是可选的。可选属性用 ? 标记，表示该属性可以存在，也可以不存在。

属性可以被声明为只读（readonly），这样一旦对象创建后该属性就不能被修改。

使用接口定义对象类型

接口（interface）用于定义对象的结构，接口通常用于描述更复杂的对象。

使用类型别名定义对象类型

类型别名（type）与接口相似，可以用来定义对象的结构。

索引签名

索引签名允许定义不确定数量的属性。常用于表示键值对结构。

元组

在 TypeScript 中，元组（Tuple） 是一种特殊的数组，允许在一个数组中存储不同类型的元素，并且每个元素的类型和位置是固定的。

元组通过指定每个元素的类型来定义。元组的元素类型和数量必须与定义时保持一致。

虽然元组的长度是固定的，但它仍然是一个数组类型，因此可以使用一些数组方法，如 push 和 pop。但要注意，push 添加的元素不会受到类型检查的限制。

访问或赋值非指定类型的位置，TypeScript 会报错。

元组支持解构赋值，可以很方便地获取元组中的元素。

元组还可以包含可选元素，即元素可以存在也可以不存在。可选元素必须放在元组的末尾，以确保类型的正确性。

枚举

在 TypeScript 中，枚举（Enum） 是一种用于定义命名常量集合的数据类型，适合表示一组固定值的场景。Enum 成员值都是只读的，不能重新赋值。

定义枚举时使用 enum 关键字，枚举中的每个成员默认从 0 开始递增编号。

可以手动指定枚举成员的数值。未赋值的成员会从最后一个成员的数值递增，但不能是 bigInt。

多个同名的 Enum 结构会自动合并，Enum 结构合并时，只允许其中一个的首成员省略初始值，否则报错；不能有同名成员，否则报错。

字符串枚举中，所有成员都必须被初始化为字符串。字符串枚举适用于对每个值有固定含义的情况。

TypeScript 支持混合字符串和数字的枚举

在数字枚举中，可以通过值来查找对应的键，称为反向映射。

any类型

有时候并不清楚一个值是什么类型，就可以用到any类型，它是一个任意类型。any 类型表示没有任何限制，该类型的变量可以赋予任意类型的值。

void类型

void 和 any 相反，any 是表示任意类型，而 void 是表示没有类型，就是什么类型都不是。JavaScript 没有空值（Void）的概念，在 TypeScript 中，可以用 void 表示没有任何返回值的函数。

void 类型的变量只能赋值为 undefined 和 null ，其他类型不能赋值给 void 类型的变量。

unknown类型

为了解决any类型“污染”其他变量的问题，TypeScript 3.0 引入了unknown类型。它与any含义相同，表示类型不确定，可能是任意类型，但是它的使用有一些限制，不像any那样自由，可以视为严格版的any。

unknown类型的变量，不能直接赋值给其他类型的变量（除了any类型和unknown类型）。

unknown类型变量的方法和属性不能被直接调用

never类型

在 TypeScript 中，never 类型表示的是永远不会有值的类型。它主要用在那些不可能正常结束的函数上，比如总是抛出错误或陷入无限循环的函数。

  1. 函数总是抛出异常：如果一个函数内部抛出异常，导致函数不会有正常返回值，那么可以将该函数的返回类型标记为 never。
  2. 无限循环的函数：如果一个函数的逻辑使得它永远不会结束（例如死循环），也可以使用 never 作为返回类型。
