TS学习-高级类型

TypeScript 的高级类型提供了更灵活的方式来操作和组合类型，使得类型系统能够处理更复杂的场景。

高级类型

联合类型

联合类型是一种互斥的类型，该类型同时表示所有可能的类型。**联合类型可以理解为多个类型的并集。**联合类型使用 | 运算符，表示一个变量可以是多个类型中的一种。联合类型常用于函数参数和返回值类型。

交叉类型

交叉类型使用 & 运算符，将多个类型合并成一个新类型。新类型具有所有被交叉类型的属性和方法。交叉类型可以理解为多个类型的交集。

类型别名

类型别名使用 type 关键字来定义一个新的类型名称，便于复用复杂类型。

索引类型

索引类型允许使用动态的属性名。

索引查询操作符

使用 keyof 操作符可以返回一个由这个类型的所有属性名组成的联合类型

索引访问操作符

索引访问操作符就是[]，其实和访问对象的某个属性值是一样的语法，但是在 TS 中它可以用来访问某个属性的类型

映射类型

映射类型可以根据一个现有类型创建一个新的类型。可以用 in 关键字对类型的每个属性进行变换。

条件类型

条件类型使用 T extends U ? X : Y 语法表示，如果 T 能够分配给 U，则返回类型 X；否则，返回类型 Y。

内置类型工具

TypeScript 提供了一些常用的内置类型工具，帮助处理常见的类型变换任务：

  1. Partial<T>：将类型 T 的所有属性变为可选。
  2. Required<T>：将类型 T 的所有属性变为必填。
  3. Readonly<T>：将类型 T 的所有属性变为只读。
  4. Pick<T, K>：从类型 T 中选择部分属性，创建新类型。
  5. Omit<T, K>：从类型 T 中排除部分属性，创建新类型。
  6. ReturnType<T>：获取函数 T 的返回值类型。
  7. Parameters<T>：获取函数 T 的参数类型。

可辨识联合类型

可辨识联合类型用于处理联合类型中的多个可能类型。可辨识联合类型通过在每个类型中添加一个唯一的 字面量类型属性，帮助 TypeScript 根据这个属性推断出联合类型的具体分支，进而在代码中进行类型缩小和类型检查。

  1. 联合类型：多个类型的组合。
  2. 字面量类型：包含一个唯一的、具备字面量值的属性。
  3. 类型缩小：可以在条件分支中通过判断该字面量属性的值来缩小类型范围。

字面量类型

字面量类型通过使用特定的字面量值（如字符串、数字、布尔值等）来约束变量的取值范围。字面量类型使得我们可以指定一个变量只能接受特定的值，从而提供更精确的类型控制。

  1. 字符串字面量类型
  2. 数字字面量类型
  3. 布尔字面量类型

字符串字面量类型

字符串字面量类型用于限制变量的值为一个或多个特定的字符串。

数字字面量类型

数字字面量类型用于限制变量的值为特定的数字。

布尔字面量类型

布尔字面量类型用于限制变量只能为 true 或 false。

模板字面量类型

在 TypeScript 4.1 版本中新增了模板字面量类型，它以字符串字面量类型为基础，可以通过联合类型扩展成多个字符串。它与 JavaScript 的模板字符串语法相同，但是只能用在类型定义中使用。

基本语法

当使用模板字面量类型时，它会替换模板中的变量，返回一个新的字符串字面量。

变量限制

模板字面量类型中的变量只允许是string、number、bigint、boolean、null、undefined或这些类型的联合类型。

This 类型

This 类型在 TypeScript 中是一种特殊的类型，它表示 当前上下文中的 this 对象的类型。通常用于方法和函数中，帮助类型系统推断和检查函数内部 this 的类型，尤其是在面向对象编程中。this 类型通常用于对象方法的上下文绑定，确保 this 的类型符合预期。

类或接口中This类型

在 JavaScript 或 TypeScript 中，this 通常引用当前对象的实例。TypeScript 通过 this 类型来明确 this 的结构和类型

使用 this 类型明确上下文

在 TypeScript 中，我们可以显式地指定 this 的类型，这在一些复杂的函数或回调中尤为重要。尤其是在回调函数中，如果没有正确绑定 this，TypeScript 可能会无法推断出 this 的类型。

使用箭头函数绑定 this

箭头函数不会创建自己的 this，它会从外部上下文中继承 this，所以可以避免在回调中 this 丢失的问题。

使用 this 类型的函数声明

除了在类中，this 类型也可以用于普通函数或方法中，显式声明 this 的类型。比如，定义一个函数时，能够准确指明函数调用时的 this 上下文。

类型推断

如果没有明确的指定类型，那么 TypeScript 会依照类型推论（Type Inference）的规则推断出一个类型。

基础类型

在变量的定义中如果没有明确指定类型，编译器会自动推断出其类型

多类型联合

当定义一个数组或元组这种包含多个元素的值时，多个元素可以有不同的类型，这时 TypeScript 会将多个类型合并起来，组成一个联合类型

上下文类型

上下文类型是根据左侧的类型推断右侧的类型

类型守卫

类型守卫是 TypeScript 提供的一种方法，用来在代码中动态缩小变量的类型范围，从而增强类型安全性。类型守卫在控制类型检查的逻辑判断时非常有用，常用于条件语句中，以确保不同的类型能够通过类型缩小得到正确的访问和处理。

typeof 类型守卫

typeof 类型保护用于确定变量的类型，它只能识别 boolean、string、bigint、symbol、undefined、function、number。

instanceof 类型守卫

用于检查某个对象是否是某个类的实例，适合于类和构造函数类型。

in 关键字类型守卫

用于检查对象是否具有特定属性，通常返回一个布尔值，指示该属性是否存在于对象中，适用于对象字面量类型。

自定义类型守卫

使用 is 关键字定义自定义的类型守卫，通常用于复杂的类型判断。类型谓词函数返回一个布尔值，TypeScript 可以在布尔判断的分支中推断具体类型。

类型断言

类型断言是 TypeScript 中的一种机制，它允许开发者明确告诉 TypeScript 编译器一个值的类型，从而绕过类型检查。类型断言不会对代码产生运行时的影响，它只是告诉编译器“相信我，我知道这个值的类型是什么”，通过这种方式可以绕过 TypeScript 默认的类型推断。

类型断言语法

TypeScript 提供了两种方式进行类型断言：

  1. 使用尖括号（<Type>）语法

  2. 使用 as 语法

类型断言条件

类型断言常见的使用场景是在你已经知道某个值的类型，但 TypeScript 无法推断出来时。

  1. 操作动态类型的值：当你从 any 类型或者从外部库中获取到某个值时，TypeScript 可能无法自动推断出其类型，这时可以使用类型断言来明确指定类型。
  2. 与 DOM 操作相关的代码：操作 DOM 元素时，TypeScript 无法知道特定元素的类型，使用类型断言可以使代码更清晰。
  3. 对象类型的细化：在处理联合类型时，某些情况下需要根据上下文对对象类型进行细化。

as const 断言

用于将一个值的类型锁定为 字面量类型，而不是它的普通类型（如 string、number、boolean 等）。通常用于对象、数组或其他常量的定义，目的是让 TypeScript 确保这些值不会被修改，并且其类型被缩小为最具体的字面量类型。

对常量值使用 as const

当我们定义一个字符串或数字常量时，TypeScript 默认会推断出更宽泛的类型，比如 string 或 number，但如果你希望将其类型限制为它的字面量类型，可以使用 as const。

对数组使用 as const

使用 as const 还可以将数组中的元素类型锁定为字面量类型，而不是常规的 string[] 或 number[] 类型。

对对象使用 as const

当对对象进行定义时，as const 可以确保对象的属性值被推断为字面量类型，而不是更宽泛的 string、number 或 boolean 类型。

as const 与 readonly 的关系

as const 不仅将值的类型缩小为字面量类型，还会将对象或数组的属性变为 readonly，从而确保这些值不可修改。

非空断言

对于那些可能为空的变量（即可能等于undefined或null），TypeScript 提供了非空断言，保证这些变量不会为空，写法是在变量名后面加上感叹号!。
