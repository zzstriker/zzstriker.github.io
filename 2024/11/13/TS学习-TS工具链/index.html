TS学习-TS工具链

TypeScript 工具链涉及 TypeScript 项目的开发、构建、调试和优化的方方面面，包括编译器、类型检查工具、代码格式化工具、构建工具、调试工具以及与 IDE 的集成等。

工具链

TypeScript 编译器 (tsc)

TypeScript 的核心工具是 tsc（TypeScript Compiler），它负责将 .ts 文件编译为 JavaScript，并进行类型检查。

tsconfig.json 配置文件

tsconfig.json 文件包含 TypeScript 编译器的配置选项。项目根目录中的 tsconfig.json 使 tsc 可以识别项目并自动处理其中文件。

常用配置项

构建工具的 TypeScript 支持

现代 JavaScript 构建工具，如 Webpack、Babel 和 Vite，都可以集成 TypeScript，通过插件来实现编译和类型检查。

Webpack

可以使用 ts-loader 或 babel-loader 来在 Webpack 项目中集成 TypeScript。

Webpack 配置示例

Babel

Babel 主要用于将 TypeScript 代码编译为 JavaScript，但不会进行类型检查。可以结合 @babel/preset-typescript 使用。

Babel 配置

Vite

Vite 是现代的构建工具，开箱即用支持 TypeScript，无需额外配置。可以直接在 vite.config.ts 中定义 TypeScript 配置。

Linting 和代码风格

ESLint

ESLint 是最流行的 JavaScript 代码检查工具，支持 TypeScript。推荐使用 @typescript-eslint 插件集成 TypeScript 支持。

ESLint 配置

Prettier

Prettier 是一个代码格式化工具，可以结合 ESLint 使用。安装 Prettier 及其 ESLint 插件
