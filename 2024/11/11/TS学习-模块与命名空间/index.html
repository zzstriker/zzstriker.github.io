TS学习-模块与命名空间

在 TypeScript 中，模块和命名空间用于组织和管理代码，避免命名冲突，使项目结构更清晰。尽管它们有相似之处，但其用法、适用场景和作用也有所不同。

模块与命名空间

模块

模块是 TypeScript 基于 ES6 模块规范实现的一种代码组织方式，具有更好的代码复用性和封装性。模块的代码会默认处于文件的私有作用域中，不会污染全局命名空间，需要显式导入和导出才能在其他文件中访问模块中的内容。

模块的定义与使用

模块在 TypeScript 中是一个独立的文件，模块中的内容（类、接口、变量等）需要通过 export 导出，才能在其他模块中被导入使用。

一个模块也可以使用 export default 来导出默认成员，导入时可以自定义名称

import type

import type 是 TypeScript 引入的一种语法，用于显式地声明仅用于类型信息的导入。

  1. 减少运行时代码：类型信息在编译时使用，而不会在运行时存在。因此，使用 import type 可以确保编译后的代码中不会包含不必要的导入语句，从而减小最终的代码体积。
  2. 避免循环依赖：在某些复杂的模块关系中，普通的导入可能会导致循环依赖问题。由于类型导入不会在运行时存在，使用 import type 可以有效避免这些问题。
  3. 明确类型用途：通过显式声明某个导入仅用于类型，代码的可读性和维护性得到提高。开发者和工具可以更清楚地理解导入的目的。

import type 与 import 的区别

import：

  1. 用于导入值（变量、函数、类等），不仅仅是类型。
  2. 会在编译后的 JavaScript 中保留对应的导入语句。

import type：

  1. 仅用于导入类型信息（接口、类型别名、枚举等）。
  2. 不会在编译后的 JavaScript 中生成导入语句。

模块的优点

  1. 模块化：模块将代码分隔到不同的文件中，便于代码管理和维护。
  2. 作用域隔离：模块内的代码默认在模块的作用域中，不会污染全局命名空间。
  3. 依赖管理：使用 import 和 export 来声明模块之间的依赖关系，编译器能检查模块的导入和导出是否正确。

命名空间

命名空间是 TypeScript 中用于组织代码的一种早期机制，它的主要目的是解决全局命名空间污染的问题。命名空间常用于将大量相关功能或类型放在同一个命名空间中，避免命名冲突。命名空间适合在浏览器环境中使用，因为命名空间的内容会直接暴露在全局作用域下。

命名空间的定义与使用

命名空间使用 namespace 关键字定义，通常用于包含一组相关的类、接口、函数或常量。命名空间可以嵌套、分散在多个文件中，但需要使用 /// <reference path="..."> 或 tsconfig.json 的 include 设置来引用关联的文件。

嵌套命名空间

命名空间可以嵌套，从而进一步组织代码。

命名空间的优点

  1. 分组相关代码：命名空间将相关代码分组在一起，有助于组织代码结构。
  2. 避免全局命名冲突：命名空间将代码封装在一个命名空间下，减少了与其他代码的命名冲突。
  3. 适合大型应用：对于大型应用程序或复杂的代码库，命名空间可以帮助将逻辑进行分层管理。
