文档结构

什么是HTML头部？

HTML 文档包含一个文档类型声明和 <html> 根元素，<html> 元素中嵌套的是<head>和<body>元素。页面在浏览器加载后<head>元素内的内容不会在浏览器中显示，它的作用是保存页面的一些元数据。

HTML文档结构

<!DOCTYPE html>

Doctype是HTML5的文档声明，通过它可以告诉浏览器，使用哪一个HTML版本标准解析文档。在浏览器发展的过程中，HTML出现过很多版本，不同的版本之间格式书写上略有差异。如果没有事先告诉浏览器，那么浏览器就不知道文档解析标准是什么？此时，大部分浏览器将开启最大兼容模式来解析网页，我们一般称为怪异模式，这不仅会降低解析效率，而且会在解析过程中产生一些难以预知的bug，所以文档声明是必须的。

<html>

<html> 元素是 HTML 文档的根元素。它是 <head> 和 <body> 的父级，其中包含 HTML 文档中除 DOCTYPE 之外的所有内容。

lang属性

用于定义文档的主要语言，lang属性的值是两个或三个字母的ISO语言代码。

lang属性不限于 <html> 标记。如果网页中有文本使用的语言与主文档的语言不同，则应使用lang属性标识文档中主要语言。

<head>元素

包含文档的所有元数据，其中必须包含标题、字符集、视口设置。

<title>

设置页面的标题，显示在浏览器标签页上，也作为收藏网页的描述文字。

<meta>

用于提供关于HTML文档的元数据。

属性：

charset：指定文档中的字符编码

content：是http-equiv或name属性的值

http-equiv：定义了一个编译指示指令，所允许的值都是特定 HTTP 标头的名称，例如：content-type、refresh

name：用于描述网页，例如：keywords、description、viewport、robots、author

<link>

设置HTML文档与外部资源的关系，常用于链接样式表，也可以创建站点图标。

属性：

rel：命名链接文档与当前文档的关系，例如：icon、stylesheet

href：指定被链接资源的URL。URL可以是绝对的，也可以是相对的

<style>

包含文档的样式信息或文档的部分内容。其中的 CSS 会应用于包含 <style> 元素的文档内容。

<script>

用于嵌入可执行代码或数据，通常用作嵌入或者引用 JavaScript 代码。

属性：

type：表示所代表的脚本类型

async：脚本并行加载，加载完成之后立即执行，执行时机不确定，有可能阻塞HTML解析

defer：脚本并行加载，等待HTML解析完成之后，按照加载顺序执行脚本

src：定义引用外部脚本的 URI，直接在文档中嵌入脚本

<noscript>

定义了一段HTML，如果页面上的脚本类型不受支持，或者浏览器当前关闭了脚本，则插入该HTML。

<base>

一个文档中只能有一个<base>元素。指定用于一个文档中包含的所有相对URL的根URL，类似提供一个指定的默认目标地址。

属性：

href：设置文档中相对URL地址的基准URL，在链接中设置相对的访问目标，浏览器便会自动解析出一个完整的链接地址进行访问，倘若链接的访问目标为空，浏览器也会根据<base>所给的URL进行访问。值得注意的是，<base>标签需放在所有包含URL地址的语句前面。

target：网页窗口的打开方式

  1. _blank：在新窗口打开链接页面。
  2. _parent：在上一级窗口中打开链接。
  3. _self：在当前窗口打开链接,此为默认值，可以省略。
  4. _top：在浏览器的整个窗口打开链接，忽略任何框架。
