CSS学习(16)-Grid

Grid 是一个二维布局系统。利用Grid，可以把内容按照行与列的格式进行排版。

Grid

网格是一组相交的水平线和垂直线，它定义了网格的列和行。

网格容器

通过在元素上声明 display：grid 来创建一个网格容器。一旦我们这样做，这个元素的所有子元素将成为网格项。

网格项

网格容器的子元素被称为网格项。

网格轨道

网格轨道是网格上任意两条相邻线之间的空间，使用 grid-template-rows 和 grid-template-column 属性定义网格上的行和列。

固定尺寸

通过设置非负的数值或者百分比，定义网格行和列的尺寸大小。

弹性尺寸

使用 fr 单位用于表示轨道尺寸配额，表示按配额比例分配可用空间。

混合尺寸

fr 和其它长度单位混合使用时，fr 的计算基于其它单位分配后的剩余可用空间。

最值

函数 minmax() 用于定义轨道最小 / 最大边界值，函数 minmax() 接收两个参数：第一个参数表示最小轨道尺寸，第二个参数表示最大轨道尺寸。长度值可以是 auto，表示轨道尺寸可以根据内容大小进行伸长或收缩。

重复

函数 repeat() 用于定义重复的网格轨道，函数 repeat() 接收两个参数：第一个参数表示重复的次数，第二个参数表示轨道尺寸。

显式和隐式网格

在创建网格时，使用 grid-template-rows 和 grid-template-column 属性显式定义网格上的行和列。由于内容太多，需要更多的网格轨道，那么网格就会在隐式网格中创建行和列。默认情况下，这些轨道会自动调整大小，因此它们的大小取决于轨道内的内容。可以使用 grid-auto-rows 和 grid-auto-column 属性为隐式网格中创建的轨道定义大小。

网格线

定义网格时，定义的是网格轨道，而不是网格线。网格布局会创建编号的网格线来定位每一个网格单元。

跨越多列

通过对 grid-column-start 和 grid-column-end 属性的设置，使该网格项跨越多列。

跨越多行

通过对 grid-row-start 和 grid-row-end 属性的设置，使该网格项跨越多行。

简写

属性 grid-row 是 grid-row-start 和 grid-row-end 的简写形式。

属性 grid-column 是 grid-column-start 和 grid-column-end 的简写形式。

  1. 如果只指定一个值，它表示 grid-row/column-start
  2. 如果两个值都指定，第一个表示 grid-row/column-start ，第二个值表示grid-row/column-end。而且必须用斜杠(/)分隔这两个值

span

关键字 span 用来指定跨越行或列的数量。

反方向计数

由 grid-template-rows 和 grid-template-column 属性显式定义的网格，右端的列线和底端的行线。这些线会被记为 -1，然后你可以从这往前数，所以倒数第 2 条线会被记为 -2。隐式定义的网格不支持反方向计数。

网格单元格

网格单元格是网格项中最小的单位，一个父级元素被定义为网格元素，那么它的子级元素将会排列在每个事先定义好的网格单元格中。

网格间距

网格单元格之间的横向间距（gutter）或纵向间距（alley）可以使用 column-gap 和 row-gap 属性或简写 gap 来创建。

网格区域

网格项可以按行或列跨越一个或多个单元格，这样就形成了一个网格区域，网格区域必须是矩形。

网格项区域

在前面已经通过 grid-row 和 grid-column属性实现网格线跨越的简写，可以通过 grid-area 属性进一步简写。grid-area 值的顺序如下：

  1. grid-row-start
  2. grid-column-start
  3. grid-row-end
  4. grid-column-end

网格模板区域

上面我们使用 grid-area 属性定义了一个网格项区域，也可以先给一个区域命名，然后在 grid-template-areas 属性值中指定这个区域的位置。

网格布局

网格布局默认是沿着行放置网格项，可以通过 grid-auto-flow 属性来指定网格如何布局。

属性值：

  1. row：逐行填充来排列元素，在必要时增加新行。
  2. column：逐列填充来排列元素，在必要时增加新列。
  3. dense：如果使用，在网格中布局元素时，后面出现了稍小的元素，会试图去填充网格中前面留下的空白，这样做会填上稍大元素留下的空白，但同时也可能导致原来出现的次序被打乱。如果省略，后面的元素不会试图填充前面的空白，保证所有的元素按照次序出现。

形式：

  1. 单个关键字：row、column，或 dense 中的一个。
  2. 两个关键字：row dense 或 column dense。
