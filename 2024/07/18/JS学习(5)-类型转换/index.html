JS学习(5)-类型转换

Javascript 是一种弱类型语言，这意味着变量是没有明确类型的，而是由 JavaScript 引擎在编译时隐式完成，类型转换是将一种数据类型转换为另一种数据类型。

类型转换

任何数据类型，无论是原始类型还是对象，都可以进行类型转换。尽管原始类型和对象的转换逻辑各不相同，但是都只能转换为三种类型：字符串（string）、数字（number）、布尔值（boolean）。

JavaScript 中的类型转换有两种方式：

  1. **隐式类型转换：**由 JavaScript 编译器完成的自动类型转换。
  2. **显式类型转换：**由开发人员完成的手动类型转换。

显式类型转换

通过 JavaScript 内置的一些 API 将一种类型转换为另一种类型，这称为显式类型转化。

String类型转换

使用String()和.toString()可以将不同类型的值转换为字符串，String()函数可以将任何类型的值转换为字符串，需要注意的是，null 和 undefined 没有 toString() 方法，直接调用会导致错误。

Number类型转换

有3个函数可以把非数值转换为数值，Number()、parseInt()和parseFloat()。Number()可以用于任何数据类型，另外两个用于把字符串转换成数值。

  1. Number()：可以用于任何数据类型

    • 如果是 Boolean 值，true 和 false 分别被转换为 1 和 0

    • 如果是 null 值，返回 0

    • 如果是 undefined，返回 NaN

    • 如果是字符串，遵循下列规则：

      • 如果字符串中只包含数字（有效的进制数值），则将其转换为十进制数值
      • 如果字符串中包含有效的浮点格式，则将其转换为对应的浮点数值
      • 如果字符串是空的，则将其转换为 0
      • 如果字符串包含除上述格式之外的字符，则将其转换为 NaN

    • 如果是对象，调用 valueOf()方法，并按照上述规则转换返回的值。如果转换结果是 NaN，则调用 toString()方法，再按照转换字符串的规则转换

  2. parseInt()：用于把字符串转换成整数

    • 如果第一个字符不是数值字符、加号或减号，返回 NaN。
    • 空字符串也会返回 NaN
    • 如果第一个字符是数值字符、加号或减号，则继续依次检测每个字符，直到字符串末尾，或碰到非数值字符

  3. parseFloat()：用于把字符串转换成浮点数

    • 如果第一个字符是数值字符、加号或减号，则继续依次检测每个字符，直到字符串末尾，或碰到无效的浮点数值字符
    • 忽略字符串开头的零
    • 十六进制数值始终会返回 0
    • 如果字符串表示整数（没有小数点或者小数点后面只有一个零），则 parseFloat()返回整数

Boolean类型转换

任何数据类型的值调用Boolean()函数，总是会返回一个 Boolean 值，至于返回的是true还是false，取决于要转换值的数据类型及其实际值，下表是各类型对应转换的规则：

数据类型 转换为true的值 转换为false的值
Boolean true false
String 任何非空的字符串 “”（空字符串）
Number 任何非零数字值（包括无穷大） 0和NaN
Object 任何对象 null
undefined 不适用 undefined

隐式类型转换

隐式类型转换是指 JavaScript 引擎在遇到不同类型的数据时自动进行类型转换，以便进行相应的操作。

加号运算符

当遇到加号运算符的时候，会按照以下规则进行转换，优先级从高到低

  1. 如果只有一侧有值，会将该值转换为Number类型
  2. 当一侧为String类型，被识别为字符串拼接，并会优先将另一侧转换为字符串类型。
  3. 当一侧为Number类型，另一侧为原始类型，则将原始类型转换为Number类型。
  4. 当一侧为Number类型，另一侧为引用类型，将引用类型和Number类型转换成字符串后拼接。

减乘除运算符

对各种非Number类型运用数学运算符(- * /)时，会先将非Number类型转换为Number类型

相等运算符

  1. NaN和其他任何类型比较永远返回false（包括NaN）
  2. Boolean 和其他任何类型比较，Boolean首先被转换为Number类型
  3. String和Number比较，先将String转换为Number类型。
  4. null == undefined比较结果是true，除此之外，null、undefined和其他任何结果的比较值都为false。
  5. 原始类型和引用类型做比较时，引用类型会依照ToPrimitive规则转换为原始类型，即遵循先valueOf后toString的模式期望得到一个原始类型，如果得不到原始类型，就会抛出 TypeError
