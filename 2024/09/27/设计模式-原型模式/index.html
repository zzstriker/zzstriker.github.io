设计模式-原型模式

原型模式是一种创建型设计模式，它允许通过复制现有对象来创建新对象，而不是通过实例化类来创建对象。通过原型模式，可以避免重复创建相似的对象，尤其是当对象的创建成本较高时，复制已有对象的效率更高。在 JavaScript 中，由于它的原型继承特性，原型模式可以非常自然地实现。

原型模式

组成

  1. 原型接口（Prototype）：定义一个可以被克隆的接口，通常提供 clone() 方法，用于复制自身。
  2. 具体原型（Concrete Prototype）：实现原型接口的具体对象，负责复制自身。
  3. 客户端（Client）：通过调用原型对象的 clone() 方法来创建新的对象，而不是直接使用 new 操作符。

代码实现

说明

  1. 原型对象 (personPrototype)：这是一个带有初始化和行为方法的原型对象，所有基于这个原型创建的对象都会继承这些方法。
  2. Object.create()：JavaScript 的内置方法，用来基于给定的原型对象创建一个新对象。新对象会继承原型对象的属性和方法。
  3. init() 方法：每个克隆对象都会调用 init() 方法进行初始化。这相当于一个构造函数，初始化对象的 name 和 age 属性。
  4. 克隆：person1 和 person2 是两个独立的对象，它们基于相同的原型创建，并且分别通过 init 方法初始化了不同的状态。

优点

  1. 减少对象创建的开销：特别是对于复杂对象，克隆比重新创建成本更低。
  2. 动态创建对象：在运行时动态创建对象，不需要提前知道具体的类型。
  3. 避免复杂类的创建：对于那些需要经过复杂初始化的对象，可以通过克隆快速得到一个已初始化的副本。

缺点

  1. 深拷贝问题：如果对象中有嵌套对象，浅拷贝可能会带来副作用，需要手动处理深拷贝。
  2. 实现复杂性：在一些编程语言中，实现深拷贝可能比较复杂，尤其是当对象包含引用其他对象时。

与其他模式的对比

  1. 原型模式 vs 工厂模式：原型模式通过复制现有对象来创建新对象，而工厂模式是通过调用工厂方法来实例化对象。原型模式适用于需要克隆的场景，而工厂模式适用于控制对象创建的场景。
  2. 原型模式 vs 单例模式：单例模式只允许一个实例，而原型模式允许通过克隆创建多个类似的对象。两者的目标不同，单例模式强调唯一性，而原型模式强调高效的对象创建。
