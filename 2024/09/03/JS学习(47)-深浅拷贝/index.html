JS学习(47)-深浅拷贝

在 JavaScript 中，深拷贝和浅拷贝是复制对象的两种不同方式。它们在处理嵌套对象（即对象内包含其他对象或数组）时有着显著的区别。

深浅拷贝

浅拷贝

浅拷贝只复制对象的第一层属性，而不会递归复制嵌套对象。换句话说，浅拷贝复制了原始对象的所有值，但对于对象或数组等引用类型，只是复制了引用，而不是实际的值。

常用的浅拷贝方法

循环遍历赋值

  1. for循环：对于数组，可以使用 for 循环来实现浅拷贝。

  2. for...in：for...in 循环可以遍历对象的所有可枚举属性并将它们复制到新的对象中。

  3. for...of：for...of 是 ES6 引入的一种遍历语句，主要用于遍历可迭代对象（如数组、字符串、Map、Set 等）。

  4. forEach：forEach 主要用于遍历数组

  5. map：map 方法通常用于创建一个新数组，其中每个元素都是原始数组中元素经过某种处理后的结果。非常适合用于数组的浅拷贝。map 方法遍历数组并返回一个新数组，元素是原始数组的浅拷贝。

对象

  1. Object.assign()：Object.assign() 常用于浅拷贝对象，它将一个或多个源对象的可枚举属性复制到目标对象中，并返回目标对象。如果某个源对象的属性是引用类型（如对象或数组），Object.assign() 只复制其引用，而不是创建新的独立对象。

  2. 拓展运算符{...obj}：对象展开运算符（...）用于将对象的所有可枚举属性浅拷贝到新的对象中。

数组

  1. concat()：用于将两个或多个数组合并为一个新数组。它可以用来实现数组的浅拷贝

  2. slice()：用于创建一个新数组，包含原数组的一个子集。它可以用于实现浅拷贝数组。

  3. Array.from()：用于从类数组对象或可迭代对象创建一个新的数组实例。

实现浅拷贝

深拷贝

深拷贝递归地复制对象的所有属性，包括嵌套的对象或数组。深拷贝创建一个与原始对象完全独立的新对象，修改新对象中的嵌套对象或数组时，不会影响原始对象。

常用的深拷贝方法

  1. JSON序列化和反序列化：将对象转换为 JSON 字符串，然后再将其转换回对象。这样可以创建一个新的对象副本，其中包括所有嵌套的属性。这个方法的缺点是它不适用于包含函数、undefined、Symbol、Map、Set、Date、RegExp、Error 等特殊对象的情况，因为这些会在序列化过程中丢失或变得不再有效。

  2. 循环递归拷贝：使用循环递归来实现深拷贝可以处理对象中的嵌套结构，并且能够处理包含循环引用的对象。
