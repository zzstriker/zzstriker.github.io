Node学习-网络编程

Node.js 提供了强大的文件系统 (File System, FS) 模块，用于对文件和目录进行操作。它既支持同步操作，也支持异步操作，能够很好地满足不同场景下的需求。

网络编程

TCP

TCP（Transmission Control Protocol，传输控制协议）是一种面向连接的、可靠的协议。它保证数据在网络中的顺序和完整性，在数据传输前需要建立连接，适合用于需要高可靠性的通信场景（如网页浏览、文件传输等）。Node.js提供了net模块来处理TCP协议。该模块允许我们创建TCP服务器和客户端，进行基于TCP的通信。

创建TCP服务器

net.createServer方法用于创建一个TCP服务器，接收来自客户端的连接请求，并为每个连接创建一个Socket对象，通过该对象进行数据交换。

  1. net.createServer() 用于创建一个 TCP 服务器，它的回调函数会在每次有客户端连接时被调用，回调函数的参数 socket 表示与客户端的连接。
  2. socket.setEncoding('utf8') 设置传输的数据的编码格式为 utf8，这样你就可以直接以字符串的形式处理传输的数据。
  3. socket.on('data', callback) 用于监听客户端发送的数据，每次客户端发送数据时，都会触发这个事件。
  4. socket.write() 用于向客户端发送数据。
  5. socket.on('end', callback) 当客户端关闭连接时触发此事件。
  6. socket.on('error', callback) 用于监听连接中的错误。

创建TCP客户端

net.connect 和 net.createConnection 方法用于创建一个TCP客户端，用于连接TCP服务器

  1. client.connect(port, host, callback) 用于连接到指定的服务器端口和地址。在这个例子中，它连接到 127.0.0.1:8080。
  2. client.write() 用于向服务器发送数据。
  3. client.on('data', callback) 用于监听从服务器接收到的数据。
  4. client.end() 关闭与服务器的连接。
  5. client.on('end', callback) 用于监听连接关闭事件。

UDP

UDP（User Datagram Protocol，用户数据报协议）是一种无连接、不可靠的协议。与TCP不同，UDP不会建立连接，它直接将数据报发送到目标地址，适合用于传输不需要高可靠性的短小数据包（如视频流、实时游戏通信等）。Node.js 使用 dgram 模块支持 UDP 编程。

创建UDP服务器

  1. dgram.createSocket('udp4')：创建一个 UDP 套接字，'udp4' 表示使用 IPv4 协议，如果你想使用 IPv6，可以选择 'udp6'。
  2. server.on('message', callback)：监听客户端发送的消息，每次接收到消息时，都会调用这个回调函数，msg 是消息内容，rinfo 包含客户端的 IP 地址和端口信息。
  3. server.send()：向客户端发送数据。rinfo.address 和 rinfo.port 获取客户端的 IP 和端口信息，以便将响应消息发回客户端。
  4. server.on('listening', callback)：监听服务器启动并绑定端口后的事件，这时可以获得服务器的地址信息。
  5. server.bind(PORT, HOST)：绑定服务器监听的端口和主机地址，在这个例子中是 127.0.0.1:12345。

创建UDP客户端

  1. dgram.createSocket('udp4')：创建一个 UDP 客户端，'udp4' 使用 IPv4 协议。
  2. client.send()：发送数据到服务器，Buffer.from() 用于将消息转化为 Buffer 类型，这样可以发送到网络上。
  3. client.on('message', callback)：监听从服务器接收到的响应消息。
  4. client.on('error', callback)：监听错误事件，处理连接过程中的错误。

HTTP

HTTP（HyperText Transfer Protocol，超文本传输协议）是Web通信的基础协议，主要用于客户端与服务器之间的请求和响应。HTTP是无状态的协议，每个请求都是独立的，不依赖于先前的请求或后续的请求。Node.js提供了http模块来实现HTTP服务器。

创建HTTP服务器

  1. http.createServer()：创建一个 HTTP 服务器。这个方法的回调函数会在每次有请求到达时被调用，req 是请求对象，res 是响应对象。
  2. res.setHeader('Content-Type', 'text/html')：设置响应头，这里我们设置了 Content-Type 为 text/html，表示响应的内容是 HTML 格式。
  3. res.statusCode = 200：设置响应的状态码为 200，表示请求成功。
  4. res.write()：向客户端发送响应的主体部分，这里发送了一个简单的 HTML <h1> 标签。
  5. res.end()：结束响应并将响应发送回客户端。
  6. server.listen(PORT, HOST)：指定服务器监听的主机和端口，这里我们指定了 127.0.0.1 和端口 3000。

HTTP请求处理

HTTP 协议支持多种请求方法（如 GET、POST、PUT、DELETE 等），可以根据请求方法来处理不同的请求。

处理URL和查询参数

Node.js通过url模块提供了对URL的解析与处理能力。使用url.parse()方法可以解析URL，获取路径和查询字符串。

HTTPS

HTTPS（HTTP Secure）则是在HTTP的基础上通过SSL/TLS协议加密，确保数据传输的安全性。Node.js通过https模块提供HTTPS支持。

创建HTTPS服务器

与 http 模块类似，https 模块提供了一个 createServer() 方法来创建服务器。与 http 模块不同，使用 https 创建的服务器需要提供 SSL/TLS 证书文件。证书文件通常包括私钥（key）、证书（cert）和可能的证书链文件（ca）。

WebSocket

WebSocket 是一种基于 TCP 的协议，它在客户端和服务器之间建立一个持久的双向通信通道。与 HTTP 协议不同，WebSocket 允许客户端和服务器之间进行全双工通信，即可以随时发送数据而不需要等待对方的请求。

创建 WebSocket 服务器

  1. const wss = new WebSocket.Server({ port: 8080 })：创建一个 WebSocket 服务器，监听 8080 端口。此服务器会接收客户端连接。
  2. wss.on('connection', callback)：每当一个客户端连接到服务器时，会触发 connection 事件，ws 是 WebSocket 客户端实例。
  3. ws.on('message', callback)：监听从客户端发送的消息，每当客户端发送消息时，会触发 message 事件。
  4. ws.send(message)：向客户端发送消息。

创建 WebSocket 客户端

  1. new WebSocket('ws://localhost:8080')：创建一个 WebSocket 客户端并连接到 ws://localhost:8080 地址。
  2. ws.on('open', callback)：连接成功时会触发 open 事件。在这个回调中，我们发送一条消息到服务器。
  3. ws.on('message', callback)：监听从服务器发送的消息。每当服务器发送消息时，触发 message 事件。
  4. ws.on('close', callback)：监听 WebSocket 连接关闭的事件。
