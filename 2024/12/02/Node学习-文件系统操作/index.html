Node学习-文件系统操作

Node.js 提供了强大的文件系统 (File System, FS) 模块，用于对文件和目录进行操作。它既支持同步操作，也支持异步操作，能够很好地满足不同场景下的需求。

同步与异步方法

Node.js中的文件系统操作大多同时提供了同步和异步两种实现方式：

  1. 同步方法（fs.readFileSync等）会阻塞线程，适合需要严格执行顺序的任务。
  2. 异步方法（fs.readFile等）通过回调函数处理结果，利用Node.js的事件驱动特性避免阻塞，适用于性能要求较高的场景。

指定位置文件读写

实现从指定位置处开始读写文件，需要使用open()或openSync()方法打开文件。

  1. path：文件路径。
  2. flags：打开文件的模式，如 'r'（读取）、'w'（写入）、'a'（追加）。
  3. mode：文件权限（可选），如 0o666。
  4. callback：包含 (err, fd) 的回调，其中 fd 是文件描述符。

读取文件

使用 fs.read 方法可以从文件的指定位置读取数据

  1. fd：文件描述符。
  2. buffer：数据将被写入的缓冲区（Buffer 对象）。
  3. offset：缓冲区的起始位置（从缓冲区的第几个字节开始写入）。
  4. length：要读取的字节数。
  5. position：文件中的起始读取位置，若为 null 则从当前位置读取。
  6. callback：回调函数，参数 (err, bytesRead, buffer)：
    • bytesRead 表示实际读取的字节数。
    • buffer 是写入数据后的缓冲区。

使用 fs.readSync 方法可以实现同步方式指定位置读取文件

写入文件

使用 fs.write 方法可以从文件的指定位置开始写入数据

  1. fd：文件描述符。
  2. buffer：要写入的数据（Buffer 对象）。
  3. offset：缓冲区的起始位置（从缓冲区的第几个字节开始读取数据写入文件）。
  4. length：要写入的字节数。
  5. position：文件中的起始写入位置，若为 null 则从当前位置写入。
  6. callback：回调函数，参数 (err, bytesWritten, buffer)：
    • bytesWritten 表示实际写入的字节数。
    • buffer 是被写入的缓冲区。

使用 fs.writeSync 方法可以实现同步方式指定位置写入文件

文件流操作

Node.js 提供了强大的流 (Stream) 模型，用于处理大文件或者数据流的读取和写入。与传统的读写文件方法（如 fs.readFile 和 fs.writeFile）不同，流操作允许我们在不将整个文件加载到内存中的情况下处理数据，非常适合大文件的处理，能够显著提高性能。

读取文件流

fs.createReadStream 用于创建一个可读流，从文件中按需读取数据。它适用于处理大文件，因为它可以分块读取文件数据，而不需要将整个文件加载到内存中。

  1. path：文件路径。

  2. options：可选的配置项：

    • encoding：指定编码格式，默认为 null，即返回 Buffer 对象。

    • start：从文件的指定位置开始读取，单位是字节。

    • end：读取的结束位置，单位是字节（包括此位置）。

写入文件流

fs.createWriteStream 用于创建一个可写流，向文件中按需写入数据。类似于读取流，写入流允许数据分块写入，避免一次性将所有数据加载到内存中。

  1. path：文件路径。

  2. options：可选配置项：

    • flags：文件操作标志（默认 'w'，写入模式）。

    • encoding：编码格式，默认为 'utf8'。

    • mode：文件权限，默认为 0o666。

    • fd：文件描述符（如果已经打开文件）。

管道流

pipe 是 Node.js 中流操作的核心方法之一，它将一个流的输出直接传递给另一个流作为输入。这种方式通常用于将读取流的内容直接写入到写入流中。

转换流

转换流（Transform）是一个特殊的流类型，可以在数据从可读流传输到可写流的过程中，对数据进行修改或转换。
