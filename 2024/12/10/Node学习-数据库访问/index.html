Node学习-数据库访问

数据库访问是现代Web开发中必不可少的一部分，而Node.js凭借其异步I/O特性，使得数据库操作更加高效。

数据库访问

数据库是应用程序数据存储和管理的核心组件。Node.js通过丰富的第三方库和驱动程序，与各类数据库进行高效交互。以下是两类数据库的特点：

  1. MongoDB：一种NoSQL数据库，采用文档型存储，数据以JSON格式保存，支持灵活的动态结构。
  2. MySQL：一种关系型数据库，使用表格结构存储数据，支持复杂的SQL查询。

MongoDB数据库

MongoDB 是一个 NoSQL 类型的文档型数据库，以其灵活的架构和高性能著称。Node.js 提供了官方的 mongodb 模块，帮助开发者与 MongoDB 数据库交互。

安装完成后，就可以在代码中引入

连接 MongoDB

MongoClient 是 MongoDB 模块的主要接口，用于连接数据库。

  1. uri：数据库连接字符串，例如：mongodb://localhost:27017。
  2. options：可选参数，用于配置连接行为，如超时、池大小等。

常用操作

在 MongoDB 中，数据以 JSON 样式的文档存储在集合（collection）中，集合属于一个数据库（database）。

选择数据库

使用 client.db(databaseName) 方法选择数据库。

选择集合(表)

使用 db.collection(collectionName) 方法选择集合(表)。如果集合(表)不存在，将自动创建。

插入数据

insertOne() 和 insertMany() 方法用于向集合中插入文档。

  1. insertOne()：插入单个文档。
  2. insertMany()：插入多个文档。

查询数据

使用 find() 和 findOne() 方法查询文档。

  1. findOne()：返回匹配的第一条文档。
  2. find()：返回所有匹配的文档（需使用游标处理）。

更新数据

updateOne() 和 updateMany() 用于更新文档。

  1. $set：设置或更新字段的值。
  2. $inc：对字段值递增。

删除数据

deleteOne() 和 deleteMany() 用于删除文档。

索引管理

索引可以提高查询效率。使用 createIndex() 方法创建索引。

聚合操作

MongoDB 提供了强大的聚合框架，用于对文档进行复杂的数据处理和分析。

aggregate() 方法接受一个管道数组，其中每个阶段执行特定的操作。

事务管理

MongoDB 支持多文档事务操作，适用于具有严格一致性要求的场景。

MySQL 数据库

MySQL 是一种流行的关系型数据库管理系统，在 Web 应用开发中广泛使用。Node.js 提供了多种与 MySQL 交互的模块，其中最常用的是 mysql 和 mysql2 模块。

安装 mysql 模块

安装 mysql2 模块

mysql2 是 mysql 的增强版本，支持现代化功能（如 Promise 和 Prepared Statements）。推荐使用 mysql2，因其性能优化和更好的 Promise 支持。
