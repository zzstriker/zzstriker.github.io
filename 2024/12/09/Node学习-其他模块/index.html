Node学习-其他模块

在Node.js中有一些重要的实用模块，这些模块虽然不如核心模块常见，但在特定场景下发挥着不可或缺的作用，为Node.js应用提供了更丰富的功能和更高的开发效率。

其他模块

dns

dns模块用于处理域名系统（DNS）的解析，包括将域名解析为IP地址、反向解析IP地址等功能。

dns 模块提供两种模式：

  1. 底层操作模式：通过 libuv 提供的 DNS 协议直接查询，跨平台，通常推荐使用，例如 dns.resolve()；
  2. 系统操作模式：通过操作系统的 DNS 解析功能查询结果，受限于系统缓存。

dns.resolve

dns.resolve：是底层模式的主要方法，将域名解析为指定类型的DNS记录。

支持的记录类型：

  • A：IPv4 地址。
  • AAAA：IPv6 地址。
  • MX：邮件交换记录。
  • CNAME：规范名称记录。
  • TXT：文本记录。
  • SRV：服务记录。
  • PTR：指针记录（反向解析）。
  • NS：名称服务器记录。
  • SOA：起始授权机构记录

dns.lookup

dns.lookup：使用操作系统解析域名，将域名解析为IP地址，返回 IPv4 和 IPv6 地址。

  • family 指定地址类型：4（IPv4）或 6（IPv6）。
  • 与 resolve() 不同，lookup() 通过操作系统的 DNS 缓存处理。

dns.reverse

dns.reverse：反向解析，将IP地址解析为域名。

dns.promises

dns.promises：使用 async/await 简化异步操作。

自定义DNS服务器

dns.setServers() 可设置自定义的 DNS 服务器列表，dns.getServers() 可获取当前的服务器列表。

os

os模块提供了与操作系统相关的信息和实用方法，例如获取CPU、内存、网络接口等。

获取操作系统信息

  1. os.platform(): 返回操作系统平台。
  2. os.release(): 返回操作系统版本。
  3. os.arch(): 返回操作系统的 CPU 架构。

获取系统资源信息

  1. os.cpus(): 返回每个逻辑 CPU 内核的信息，包括型号、速度等。
  2. os.totalmem(): 返回系统总内存（单位：字节）。
  3. os.freemem(): 返回系统空闲内存（单位：字节）。
  4. os.loadavg(): 返回过去 1、5、15 分钟的系统平均负载（仅 Linux 和 macOS 支持）。

用户信息

  1. os.userInfo(): 返回当前用户的信息，包括用户名、主目录路径等。

系统运行时间

  1. os.uptime(): 返回系统运行时间（单位：秒）

网络接口信息

  1. os.networkInterfaces(): 返回系统的网络接口信息。

临时目录路径

  1. os.tmpdir(): 返回操作系统的默认临时文件目录。

系统行尾标识

  1. os.EOL: 返回操作系统的行尾标识（如 \n 在 Linux/macOS，\r\n 在 Windows）。

readline

readline模块专门用于从可读流（如 process.stdin）逐行读取输入，并对输入进行处理。它在构建交互式命令行应用程序、处理逐行输入的文件时非常有用。

创建接口

使用 readline.createInterface() 方法创建一个 readline.Interface 实例，用于管理输入输出流。

  1. input：必需，可读流，例如 process.stdin。
  2. output：可选，可写流，用于输出提示信息，例如 process.stdout。
  3. prompt：可选，自定义提示符。

核心方法

question()

用于向用户显示问题，并读取用户的输入。

  1. 回调函数的参数 answer 是用户输入的内容。
  2. rl.close() 用于关闭接口，释放资源。

on() 事件监听

readline.Interface 支持多种事件监听，例如：

  1. line：当用户输入一行并按下回车时触发。
  2. close：当接口关闭时触发。
  3. SIGINT：用户按下 Ctrl+C 时触发。

setPrompt()和prompt()

  1. setPrompt()：设置或更改提示符。
  2. prompt()：显示提示符。

pause() 和 resume()

  1. pause()：暂停 input 流。
  2. resume()：恢复 input 流。

逐行读取文件

readline 可以配合 fs 模块，用于逐行读取文件。

util

util模块提供了一组常用的实用函数，主要用于调试、继承、类型判断和格式化操作。

格式化输出

util.format()将格式化字符串与占位符结合，生成格式化的字符串。支持的占位符包括：

  1. %s：字符串
  2. %d：数字
  3. %j：JSON
  4. %%：单个百分号

调试日志

util.debuglog()生成一个调试日志函数，仅当环境变量 NODE_DEBUG 中包含指定的标识时，日志才会输出。

运行时设置环境变量

类型判断

util.types 提供了一组函数用于精确判断变量类型，例如：

  1. util.types.isPromise(value)：判断是否是 Promise。
  2. util.types.isRegExp(value)：判断是否是正则表达式。

继承与类的扩展

util.inherits()用于实现面向原型的继承，主要适用于 ES5 的代码。对于现代代码，更推荐使用 class 和 extends。

自定义对象的字符串表示

util.inspect()将对象转为字符串表示，常用于调试和日志记录。

  • depth：指定对象递归展开的层级。
  • colors：是否为输出添加颜色。

如果对象定义了 inspect 方法，util.inspect() 会调用该方法

回调转 Promise

util.promisify()将基于回调的函数转换为返回 Promise 的函数。

生成异步函数

util.callbackify()将返回 Promise 的异步函数转换为使用回调的函数。
