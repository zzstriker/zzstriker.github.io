Node学习-进程

Node.js 作为一种单线程、事件驱动的异步框架，在高并发和高性能的场景下表现出色。然而，Node.js 在处理 CPU 密集型任务时会遇到瓶颈，因为它只有一个主线程来执行所有任务。为了解决这个问题，Node.js 提供了多进程的机制，使得我们能够充分利用多核 CPU 的优势，提升应用程序的处理能力和性能。

进程

Node.js 的运行机制本质上是单线程的，它使用事件循环来处理异步任务，这种模式使得 Node.js 在 I/O 密集型应用中表现非常出色。但是，Node.js 在处理 CPU 密集型任务时，单线程会遇到性能瓶颈，无法有效利用多核 CPU 资源。

多进程架构的必要性

  1. 单核问题：Node.js 只有一个主线程，所有的任务（包括 I/O 和计算密集型任务）都在这个线程中处理。如果执行了一个复杂的计算任务，事件循环会被阻塞，导致系统无法响应其他请求。
  2. 多核优势：现代 CPU 大多数都是多核的，利用多进程可以将任务分配到多个 CPU 核心上，从而提升处理能力。

Node进程对象

Node.js通过全局对象process提供了对当前进程的管理功能。在 Node.js 中，process 对象是一个全局对象，无需 require 即可使用，它提供了一组用于与当前 Node.js 进程交互的方法和属性。

常用的process属性：

  1. process.pid：当前进程的进程ID。
  2. process.argv：一个数组，包含了启动Node.js进程时传递给程序的命令行参数。
    • 第一个元素是 Node.js 可执行文件路径。
    • 第二个元素是当前脚本文件路径。
    • 第三个元素开始是用户传递的参数。
  3. process.env：获取与当前进程相关的环境变量。
  4. process.cwd()：获取当前工作目录。
  5. process.chdir(path)：改变当前工作目录。
  6. rocess.memoryUsage()：返回一个对象，表示进程的内存使用情况
    • rss：驻留集大小。
    • heapTotal：V8 堆中总的内存量。
    • heapUsed：V8 堆中已使用的内存量。
    • external：绑定到 JavaScript 的 C++ 对象占用的内存量。
  7. process.uptime()：返回 Node.js 进程已运行的秒数。

常用的process方法：

  1. process.exit()：用于退出当前进程，通常传入参数0表示正常退出，非0表示异常退出。
  2. process.kill(pid, [signal])：向指定 PID 的进程发送信号（默认 SIGTERM）。
  3. process.nextTick(callback)：将一个回调函数放入下一次事件循环的顶部，而不是底部。它优先于异步任务（如 setTimeout、Promise）。
  4. process.stdout：标准输出流，用于输出信息。
  5. process.stderr：标准错误流，用于输出错误信息。

进程生命周期：

process 是 EventEmitter 的实例，Node.js的进程从启动到退出，都有一个生命周期。通过监听process的事件，可以对进程的生命周期进行管理。

  1. exit事件：当进程退出时触发。
  2. uncaughtException事件：捕获未处理的异常。
  3. SIGINT 事件：当用户按下 Ctrl+C 时触发
  4. beforeExit 事件：当事件循环为空、程序即将退出时触发。不同于 exit，可以在 beforeExit 中执行异步操作。

子进程的使用

尽管主进程是单线程的，Node.js 提供了创建子进程的能力，以处理 CPU 密集型任务或运行独立的脚本。

child_process 模块

child_process模块提供了创建和管理子进程的API。常用的子进程创建方法包括：

  1. spawn：适用于启动子进程并与之交互，返回子进程的流对象。

  2. fork：创建一个新的Node.js进程，适用于Node.js应用之间的通信。

  3. exec：执行一个shell命令并返回结果，适合一次性任务。

  4. execFile：直接执行一个可执行文件，避免shell的干扰，适合执行外部程序。

cluster 模块

cluster 模块是 Node.js 提供的一种多进程实现方式，它允许我们创建一个主进程和多个工作进程，从而将 HTTP 请求负载均衡地分配给多个进程，充分利用多核 CPU。

  1. 主进程和工作进程：主进程负责创建工作进程并管理它们，工作进程则负责处理具体的请求。每个工作进程都是独立的，它们共享同一个端口，但处理的请求是独立的。
  2. 工作进程的创建：通过 cluster.fork() 方法创建多个子进程，每个进程监听相同的端口。
  3. 进程通信：主进程和工作进程之间可以通过 process.send() 和 worker.on('message', ...) 方法进行通信。

进程间的消息传递

进程间通信（IPC）是指在不同进程间传递数据，主进程和子进程之间需要通过 IPC（进程间通信）进行数据交换。

child_process 模块

  1. 使用fork创建的子进程有一个内建的消息通道，可以通过process.send()发送消息，主进程通过message事件接收消息。
  2. spawn也可以与子进程进行通信，但需要通过标准输入输出（stdin/stdout）来交换数据。

cluster 模块

在 cluster 模块中，主进程和工作进程之间可以使用 process.send() 来发送消息，工作进程通过监听 message 事件来接收消息。

进程管理工具

Node.js 提供了多种工具来管理和监控进程：

  1. PM2：PM2 是 Node.js 的进程管理器，它可以帮助你管理 Node.js 的多进程应用，进行自动重启、日志记录、负载均衡等操作。
  2. process 模块：process 模块提供了有关当前进程的信息。
  3. Docker：结合容器技术，将 Node.js 应用与其依赖封装起来，便于部署和扩展。
