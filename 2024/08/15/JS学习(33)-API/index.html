JS学习(33)-API

JavaScript API 指的是浏览器和 JavaScript 提供的一系列接口（Application Programming Interface），这些接口允许开发者在网页中实现各种功能，比如操作 DOM、与服务器通信、存储数据等。

JavaScript API

跨上下文消息

跨文档消息是一种在不同执行上下文间传递信息的能力，简称为XDM（cross-document messaging），XDM的核心是postMessage()方法。

  1. postMessage()：允许不同来源（即不同域名、端口或协议）的文档彼此发送消息的安全方式。

    • message：要发送的数据，可以是字符串、对象、数组等。它会被序列化成字符串并发送。
    • targetOrigin：预期的目标窗口的来源（origin），通常包括协议、域名、端口，格式如 "https://example.com"。使用 "*" 表示不限制来源，但这在安全性上存在风险。
    • transfer（可选）：一组 Transferable 对象的列表。这个参数允许你将数据的所有权从发送者转移到接收者

File API

File API 是一组 Web API，允许 JavaScript 在客户端浏览器中处理用户文件。这些 API 提供了读取、选择和操作文件的能力。

File 类型

HTML5 在 DOM 上为文件输入元素添加了 files 集合，当用户在文件字段中选择一个或多个文件时，这个 files 集合中会包含一组 File 对象，表示被选中的文件。

  1. name：本地系统中的文件名
  2. size：以字节计的文件大小
  3. name：包含文件的MIME类型的字符串
  4. lastModifiedDate：表示文件最后修改时间的字符串

FIleReader 类型

FileReader类型表示一种异步文件读取机制，用于从文件系统读取文件。

方法

  1. readAsText(file, encoding)：从文件中读取纯文本内容并保存在 result 属性中。第二个 参数表示编码，是可选的
  2. readAsDataURL(file)：读取文件并将内容的数据 URI 保存在 result 属性中
  3. readAsArrayBuffer(file)：读取文件并将每个字符的二进制数据保存在 result 属性中
  4. readAsBinaryString(file)：：读取文件并将文件内容以 ArrayBuffer 形式保存在 result 属性

事件

  1. onload：当读取操作完成时触发
  2. onerror：当读取操作出错时触发
  3. onprogress：当读取操作进行时持续触发，用于监控读取进度

Blob

Blob 表示二进制大对象（binary larget object），是 JavaScript 对不可修改二进制数据的封装类型。

创建Blob

new Blob(blobParts, options): 通过一个数组 blobParts（包含字符串、ArrayBuffer 等）和可选的 options 对象创建一个新的 Blob 对象。

数据切分

Blob 对象有一个 size 属性和一个 type 属性，还有一个 slice()方法用于进一步切分数据。
