JS学习(27)-代理和反射

代理和反射为开发者提供了拦截并向基本操作嵌入额外行为的能力。可以给目标对象定义一个关联的代理对象，而这个代理对象可以作为抽象的目标对象来使用。

代理

代理是目标对象的抽象，可以用作目标对象的替身，但又完全独立于目标对象。目标对象既可以直接被操作，也可以通过代理来操作。直接操作目标对象会绕过代理对象施予的行为。

构造函数

  1. target：要使用 Proxy 包装的目标对象（可以是任何类型的对象，包括原生数组，函数，甚至另一个代理）
  2. handler：一个通常以函数作为属性的对象，各属性中的函数分别定义了在执行各种操作时代理 proxy 的行为

代理捕获器

使用代理的主要目的是可以定义捕获器，捕获器就是在处理程序对象中定义的“基本操作的拦截器”。每个处理程序对象可以包含零个或多个捕获器，每个捕获器都对应一种基本操作，可以直接或间接在代理对象上调用。每次在代理对象上调用这些基本操作时，代理可以在这些操作传播到目标对象之前先调用捕获器函数，从而拦截并修改相应的行为。

捕获器不变式

使用捕获器几乎可以改变所有基本方法的行为，但也不是没有限制。每个捕获的方法都知道目标对象上下文、捕获函数签名，而捕获处理程序的行为必须遵循“捕获器不变式”，防止捕获器定义出现过于反常的行为。

get()

get()捕获器会在获取属性值的操作中被调用。

  1. 返回值：返回值无限制
  2. 拦截的操作：proxy.property、proxy[property]、Object.create(proxy)[property]
  3. 捕获器参数：
    • target：目标对象
    • property：引用的目标对象上的字符串键属性
    • receiver：代理对象或继承代理对象的对象

set()

set()捕获器会在设置属性值的操作中被调用。

  1. 返回值：返回 true 表示成功；返回 false 表示失败
  2. 拦截的操作：proxy.property = value、proxy[property] = value、Object.create(proxy)[property] = value
  3. 捕获器参数：
    • target：目标对象
    • property：引用的目标对象上的字符串键属性
    • value：要赋给属性的值
    • receiver：接收最初赋值的对象

has()

has()捕获器会在 in 操作符中被调用。

  1. 返回值：返回布尔值，表示属性是否存在。
  2. 拦截的操作：property in proxy、property in Object.create(proxy)
  3. 捕获器参数：
    • target：目标对象
    • property：引用的目标对象上的字符串键属性

defineProperty()

defineProperty()捕获器会在Object.defineProperty()中被调用。

  1. 返回值：返回布尔值，表示属性是否成功定义。
  2. 拦截的操作：Object.defineProperty(proxy, property, descriptor)
  3. 捕获器参数：
    • target：目标对象
    • property：引用的目标对象上的字符串键属性
    • descriptor：包含可选的 enumerable、configurable、writable、value、get 和 set 定义的对象

getOwnPropertyDescriptor()

getOwnPropertyDescriptor()捕获器会在Object.getOwnPropertyDescriptor()中被调用。

  1. 返回值：返回对象，或者在属性不存在时返回 undefined。
  2. 拦截的操作：Object.getOwnPropertyDescriptor(proxy, property)
  3. 捕获器参数：
    • target：目标对象
    • property：引用的目标对象上的字符串键属性

deleteProperty()

deleteProperty()捕获器会在 delete 操作符中被调用。

  1. 返回值：返回布尔值，表示删除属性是否成功。
  2. 拦截的操作：delete proxy.property、delete proxy[property]
  3. 捕获器参数：
    • target：目标对象
    • property：引用的目标对象上的字符串键属性

ownKeys()

ownKeys()捕获器会在Object.keys()及类似方法中被调用。

  1. 返回值：返回包含字符串或符号的可枚举对象
  2. 拦截的操作：Object.getOwnPropertyNames(proxy)、Object.getOwnPropertySymbols(proxy)、Object.keys(proxy)
  3. 捕获器参数：
    • target：目标对象

getPrototypeOf()

getPrototypeOf()捕获器会在Object.getPrototypeOf()中被调用。

  1. 返回值：返回对象或 null
  2. 拦截的操作：Object.getPrototypeOf(proxy)、proxy.__proto__、Object.prototype.isPrototypeOf(proxy)、proxy instanceof Object
  3. 捕获器参数：
    • target：目标对象

setPrototypeOf()

setPrototypeOf()捕获器会在Object.setPrototypeOf()中被调用。

  1. 返回值：返回布尔值，表示原型赋值是否成功。
  2. 拦截的操作：Object.setPrototypeOf(proxy)
  3. 捕获器参数：
    • target：目标对象
    • prototype：target 的替代原型，如果是顶级原型则为 null。

isExtensible()

isExtensible()捕获器会在Object.isExtensible()中被调用。

  1. 返回值：返回布尔值，表示 target 是否可扩展。
  2. 拦截的操作：Object.isExtensible(proxy)
  3. 捕获器参数：
    • target：目标对象

preventExtensions()

preventExtensions()捕获器会在Object.preventExtensions()中被调用。

  1. 返回值：返回布尔值，表示 target 是否已经不可扩展。
  2. 拦截的操作：Object.preventExtensions(proxy)
  3. 捕获器参数：
    • target：目标对象

apply()

apply()捕获器会在调用函数时中被调用。

  1. 返回值：返回值无限制
  2. 拦截的操作：proxy(...argumentsList)、Function.prototype.apply(thisArg, argumentsList)、Function.prototype.call(thisArg, ...argumentsList)
  3. 捕获器参数：
    • target：目标对象
    • thisArg：调用函数时的 this 参数。
    • argumentsList：调用函数时的参数列表

construct()

construct()捕获器会在 new 操作符中被调用。

  1. 返回值：返回一个对象
  2. 拦截的操作：new proxy(...argumentsList)
  3. 捕获器参数：
    • target：目标构造函数
    • argumentsList：传给目标构造函数的参数列表
    • newTarget：最初被调用的构造函数

可撤销代理

有时候需要中断代理对象与目标对象之间的联系。对于使用 new Proxy()创建的普通代理来说，这种联系会在代理对象的生命周期内一直持续存在。 Proxy 也暴露了revocable()方法，这个方法支持撤销代理对象与目标对象的关联。撤销代理的操作是不可逆的。

反射

Reflect 是一个内置对象，提供了一些静态方法来对对象进行常见的低级操作。这些操作与 Proxy 中的捕获器方法几乎一致。通过 Reflect，可以更轻松地在 Proxy 的捕获器中调用默认行为，而不需要直接调用原始对象。

代理模式

跟踪属性访问

通过捕获 get、set 和 has 等操作，可以知道对象属性什么时候被访问、被查询。把实现相应捕获器的某个对象代理放到应用中，可以监控这个对象何时在何处被访问过

隐藏属性

代理的内部实现对外部代码是不可见的，因此可以隐藏目标对象上的属性

属性验证

所有赋值操作都会触发set()捕获器，所以可以根据所赋的值决定是允许还是拒绝赋值

参数验证

也可对函数和构造函数参数进行审查，让函数只接收某种类型的值

数据绑定

将被代理的类绑定到一个全局实例集合，让所有创建的实例都被添加到这个集合中

可观察对象

以把集合绑定到一个事件分派程序，每次插入新实例时都会发送消息
