JS学习(20)-Map

Map 是一种用于存储键值对的数据结构，类似于对象（Object），但具有一些显著的区别和优势。

Map

Map 对象是键值对的集合。Map 中的一个键只能出现一次；键在 Map 的集合中是独一无二的。Map 对象按键值对迭代，在每次迭代后会返回一个形式为 [key, value] 的数组。迭代按插入顺序进行。

构造函数

  1. iterable（可选）：一个元素是键值对的数组或其他可迭代对象。

实例属性

  1. size：返回 Map 中元素的数量

实例方法

  1. set()：向 Map 对象添加或更新一个指定的键值对

  2. get()：返回该 Map 中的指定元素，如果键相关联的值是一个对象，那么将获得该对象的引用，对该对象所做的任何更改都会有效地在 Map 对象中修改它。

  3. has()：返回一个布尔值，指示具有指定键的元素是否存在

  4. delete()：从该 Map 中删除指定键的元素

  5. clear()：移除该 Map 中的所有元素。

  6. entries()：返回一个新的 Map 迭代器对象，该对象包含了此 Map 中的每个元素的 [key, value] 对，按插入顺序排列。

  7. forEach()：按插入顺序对该 Map 中的每个键/值对执行一次提供的函数

    • callbackFn：为 map 中每个元素执行的函数。使用以下参数调用该函数
      • value：每个迭代的值
      • key：每个迭代的键
      • map：正在迭代的 Map
    • thisArg：执行 callbackFn 时用作 this 的值

  8. keys()：返回一个新的 Map 迭代器对象，该对象包含了此 Map 中的每个元素的键，按插入顺序排列。

  9. values()：返回一个新的 Map 迭代器对象，该对象包含了此 Map 中的每个元素的值，按插入顺序排列。

Map和Object对比

Map Object
默认键 Map 默认不包含任何键，只显式存入键值对 Object 有原型，包含默认的键，原型键名可能会与自己的键名冲突
键的类型 Map 的键可以为任何类型的值 Object 的键必须为 String 或 Symbol类型
键的顺序 Map 会维护键值对的插入顺序，可以根据插入顺序执行迭代操作 普通的 Object 的键是有序的，但情况并非总是如此，并且其排序比较复杂的
计算大小 Map 可以通过 size 属性获取 Map 的大小 Object 通常获取 Object.keys() 返回的数组长度来计算大小
操作性能 在涉及频繁添加和删除键值对的场景中表现更好。 未针对频繁添加和删除键值对进行优化。

WeakMap

WeakMap 是 JavaScript 中一种专门用于存储对象键值对的集合。WeakMap 是 Map 的“兄弟”类型，其 API 也是 Map 的子集。

特性

  1. 键必须是对象或 Symbol，值可以是任意的数据类型

  2. 如果键中的对象只被 WeakMap引用，没有其他引用，那么这个对象会被垃圾回收，只要键没有被垃圾回收，键值对就会存在于WeakMap中

  3. 因为 WeakMap 中的键值对有可能被垃圾回收，所以WeakMap不可迭代

构造函数

  1. iterable（可选）：一个元素是键值对的数组或其他可迭代对象。

实例方法

  1. set()：向 WeakMap 对象添加或更新一个指定的键值对

  2. get()：返回该 WeakMap中的指定元素，如果找不到该键，则返回 undefined。

  3. has()：返回一个布尔值，指示具有指定键的元素是否存在

  4. delete()：从该 WeakMap 中删除指定键的元素
