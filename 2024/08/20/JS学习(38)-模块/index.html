JS学习(38)-模块

JavaScript 模块是一种将代码分割为独立文件和模块的机制，每个模块只负责特定的功能。模块化的代码更易于维护、测试和重用。在现代 JavaScript 开发中，模块是组织代码的核心部分，尤其是在构建大型应用程序时。

模块

CommonJS

CommonJS 是一种模块化规范，最初是为服务器端 JavaScript 应用程序设计的，尤其是在 Node.js 中得到了广泛应用。它通过定义模块的格式和加载方式，帮助开发者将代码组织为独立、可复用的模块。

核心概念

  1. 模块导出 (module.exports)：在 CommonJS 中，每个文件都是一个模块，通过module.exports 将模块中的函数、对象或变量导出，以便在其他模块中使用。module.exports 是一个对象，默认情况下，模块导出的是一个空对象 {}，可以将任意内容赋值给它。

  2. 模块导入 (require)：require() 是用于加载其他模块的函数。它接受模块的路径或名称作为参数，并返回 module.exports 导出的内容。使用 require() 导入模块时，Node.js 会同步读取并执行该模块的代码，并缓存导出的结果

特点

  1. 同步加载：require() 是同步的，当调用 require() 时，Node.js 会立即加载并执行模块，直到模块的代码全部执行完毕后才继续执行后续代码。
  2. 每个文件都是模块：每个 JavaScript 文件都被视为一个独立的模块。模块内的变量和函数默认是私有的，除非通过 module.exports导出它们。
  3. 模块路径解析：require() 可以使用相对路径或绝对路径来加载模块。相对路径通常以 ./ 开头，表示当前目录下的模块。
  4. 模块缓存：模块在第一次加载后会被缓存。同一个模块多次使用 require() 只会加载一次，之后从缓存中读取。有助于提高性能。

AMD

AMD（Asynchronous Module Definition）是一种 JavaScript 模块化规范，主要用于在浏览器中异步加载模块。它的设计目的是解决浏览器环境中的模块加载问题，使得可以并行加载多个模块，避免阻塞页面渲染。

核心概念

  1. 定义模块：使用 define() 函数来定义模块。define() 接受模块名、依赖数组和工厂函数作为参数。工厂函数的返回值就是模块导出的内容

  2. 加载模块：使用 require() 函数来加载模块并使用它们。require() 也接受依赖数组和一个回调函数，回调函数在依赖项加载完成后执行

  3. 带有依赖的模块：如果一个模块依赖其他模块，可以在定义时将依赖模块的名称作为数组传递给 define()

特点

  1. 异步加载：AMD支持异步加载，避免了页面在等待 JavaScript 加载时的阻塞现象，提高了页面性能。
  2. 依赖管理：AMD 可以显式地声明模块的依赖项，使模块加载顺序更加清晰，确保了在模块使用之前，它的依赖项已经加载完毕。

UMD

UMD（Universal Module Definition）是一种 JavaScript 模块化规范，它的设计目标是让同一个模块能够在多种环境中使用，包括浏览器、Node.js 以及 AMD 加载器（如 RequireJS）等。这种通用性使得 UMD 非常适合用于需要跨平台兼容的 JavaScript 库和模块。

工作原理

UMD 会先检测当前环境支持哪种模块系统，把所有逻辑包装在一个立即调用的函数表达式，并使用对应的方式导出模块。如果没有检测到任何模块系统，它会将模块挂载到全局对象上。

ESM

ESM（ECMAScript Modules，即 ES 模块）是 JavaScript 的官方模块系统，是 ES6 中引入的一项标准。

导出模块内容

ESM 使用 export 关键字来导出模块内容。有两种导出方式：命名导出和默认导出。

  1. 命名导出：在一个模块中导出多个命名变量或函数

  2. 默认导出：默认导出用于导出一个主要的函数、对象或类。每个模块只能有一个默认导出

导入模块内容

ESM 使用 import 关键字来导入模块内容。

  1. 导入命名导出：使用大括号 {} 导入命名导出的内容

  2. 导入默认导出：默认导出可以直接导入，不需要使用大括号

  3. 导入所有内容：使用 * as 语法将模块中的所有导出内容导入为一个命名空间对象

  4. 动态导入：ESM 支持动态导入，允许在运行时按需加载模块。这通常用于条件加载或代码分割

特点

  1. 静态结构：ESM 的模块结构在编译时就确定了。JavaScript 引擎在加载模块时可以直接解析和链接模块，而不需要执行代码。
  2. 异步加载：ESM 在浏览器中是异步加载的，使用 <script type="module"> 标签来加载模块。即使模块是异步加载的，依赖关系仍然可以被正确解析。
  3. 按值导出：模块的导出是按值导出的，而不是按引用。这意味着导出的值是动态绑定的，导入模块时可以随时获取到最新的值。
  4. 作用域：每个模块都有自己的独立作用域，模块内部的变量和函数不会污染全局作用域。
