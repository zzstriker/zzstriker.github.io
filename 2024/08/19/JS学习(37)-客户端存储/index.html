JS学习(37)-客户端存储

客户端存储指的是在用户的浏览器中存储数据的各种方法。这些技术允许Web应用程序在客户端持久化数据，从而在无需与服务器交互的情况下访问和使用这些数据。

客户端存储

cookie

cookie最初用于在客户端存储会话信息，服务器在响应 HTTP 请求时，通过发送 Set-Cookie HTTP 头部包含会话信息，浏览器会存储这些会话信息，并在之后的每个请求中都会通过 HTTP 头部 cookie 再将它们发回服务器。

  1. 每个 cookie 的大小限制为约 4KB
  2. 每个域名下的 cookie 数量有限，一般最多 20-50 个

cookie 的构成

  1. 名称：唯一标识 cookie 的名称。cookie 名不区分大小写
  2. 值：存储在 cookie 里的字符串值
  3. 域：cookie 有效的域。发送到这个域的所有请求都会包含对应的 cookie
  4. 路径：请求 URL 中包含这个路径才会把 cookie 发送到服务器
  5. 过期时间：表示何时删除 cookie 的时间戳。默认情况下， 浏览器会话结束后会删除所有 cookie。
  6. 安全标志：设置之后，只在使用 SSL 安全连接的情况下才会把 cookie 发送到服务器。

Web Storage

Web Storage 是一种浏览器提供的客户端存储机制，用于在用户的浏览器中存储数据。提供了更大的存储空间和更简单的 API，且数据不会自动随 HTTP 请求发送到服务器。Web Storage 包含两种存储方式：localStorage 和 sessionStorage。

Storage类型

Storage 类型用于保存名/值对数据，直至存储空间上限。

  1. clear()：删除所有值；
  2. getItem(name)：取给定 name 的值
  3. key(index)：取给定数值位置的名称
  4. removeItem(name)：删除给定 name 的键值对
  5. setItem(name, value)：设置给定 name 的值

sessionStorage

sessionStorage 用于持久化存储数据，但数据仅在当前浏览器会话期间有效。当用户关闭浏览器标签页或窗口时，存储的数据会被删除。

  1. 会话性：数据只在当前会话（即浏览器窗口或标签页）中有效，关闭会话后数据将被清除
  2. 独立性：每个标签页或窗口有各自独立的 sessionStorage 数据，即使在同一域名下，标签页间的数据也不共享
  3. 存储空间：通常每个域名约 5MB

localStorage

localStorage 用于持久化存储数据，除非被显式删除，否则数据不会过期。即使用户关闭浏览器或计算机，数据仍然会保留。

  1. 持久性：数据在浏览器会话之间持久保存，除非用户清除浏览器数据或通过代码删除。
  2. 存储空间：通常每个域名可以存储约 5MB 数据
  3. 数据类型：只能存储字符串，其他类型需要序列化为字符串后才能存储

IndexedDB

IndexedDB 是一种在用户的浏览器中存储大量结构化数据的客户端数据库技术。用于在Web应用程序中本地存储数据。IndexedDB支持事务性存储，它允许你存储几乎任何类型的数据，包括文件、blob、数组、对象等等，且容量较大（通常为几百MB或更多），可以用于离线存储和缓存。

  1. 键值对存储：IndexedDB是一个基于对象存储(Object Store)的键值对数据库，每个数据库可以包含多个对象存储。
  2. 异步操作：IndexedDB 的所有操作都是异步的，这意味着你需要使用回调函数或 Promise 来处理查询结果。
  3. 事务支持：所有的读写操作都必须在事务内进行，事务确保了操作的原子性和一致性。
  4. 查询索引：IndexedDB支持为对象存储中的字段创建索引，从而可以加快查询速度。

对象存储

  1. open()：传入一个要打开的数据库名称。如果给定名称的数据库已存在，则会发送一个打开它的请求；如果不存在，则会发送创建并打开这个数据库的请求。这个方法会返回 IDBRequest 的实例，可以在这个实例上添加 onerror 和 onsuccess 事件处理程序。

  2. createObjectStore()：open()操作会创建一个新数据库，然后触发 upgradeneeded 事件。可以为这个事件设置处理程序，并在处理程序中创建数据库模式。如果数据库存在，而你指定了一个升级版的版本号，则会立即触发 upgradeneeded 事件，因而可以在事件处理程序中更新数据库模式。

  3. deleteObjectStore()：用于在数据库模式升级时删除不再需要的对象存储（Object Store）。这个操作通常在 onupgradeneeded 事件中执行，当你需要删除某个对象存储以便更新数据库结构时使用。

事务

事务通过调用数据库对象的transaction()方法创建。任何时候，只要想要读取或修改数据，都要通过事务来操作。

操作对象

  1. get()：获取对象
  2. add()：添加对象
  3. put()：更新对象
  4. delete()：删除对象
  5. clear()：删除所有对象

游标

使用事务可以通过一个已知键取得一条记录。如果想取得多条数据，则需要在事务中创建一个游标。 游标是一个指向结果集的指针，需要在对象存储上调用openCursor()方法创建游标。

  1. openCursor()方法也返回一个请求，须添加 onsuccess 和 onerror 事件处理程序。游标可以以升序或降序（next、prev）进行遍历。在调用 onsuccess 事件处理程序时，可以通过 event.target.result 访问对象存储中的下一条记录，如果没有更多记录为 null。

  2. update(value) 方法使用指定的对象更新当前游标对应的值，调用 update()会创建一个新请求，因此如果想知道结果，需要为 onsuccess 和 onerror 赋值

  3. 调用delelte()来删除游标位置的记录，与update()一样，这也会创建一个请求

  4. 使用continue(key)方法移动到结果集中的下一条记录。参数 key 是可选的。如果没有指定 key，游标就移动到下一条记录；如果指定了，则游标移动到指定的键

  5. 使用advance(count)方法使游标向前移动指定的 count 条记录

  6. 通过键范围来限制游标遍历的范围，only(value)方法只获取键为 value 的值，bound()方法获取结果集上限和下限的值

索引

创建索引

调用createIndex(indexName, keyPath, options)创建索引

  • indexName：是索引的名称
  • keyPath：是要索引的字段或字段数组（用于复合索引）
  • options：是可选参数，如 { unique: true }，表示该索引值必须唯一

查询数据

  1. index.get(key)：根据索引查找单条记录。
  2. index.getAll(key)：根据索引查找所有匹配的记录。
  3. index.openCursor(range)：使用游标遍历索引中的记录。
