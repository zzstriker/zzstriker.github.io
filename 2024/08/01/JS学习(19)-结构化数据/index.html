JS学习(19)-结构化数据

ArrayBuffer 和 SharedArrayBuffer 对象表示一段二进制数据，用来模拟内存里面的数据。通过这个对象，JavaScript 可以读写二进制数据。

结构化数据

ArrayBuffer

ArrayBuffer表示通用的原始二进制数据缓冲区，它是一个字节数组。不能直接操作 ArrayBuffer 中的内容，需要通过TypedArray或DataView对象来操作。

构造函数

  1. length：要创建的数组缓冲区的大小（以字节为单位）
  2. options（可选）：一个对象，可以包含maxByteLength属性，表示数据缓冲区可以调整的最大大小，以字节为单位

静态方法

  1. isView()：判断传入值是否是 ArrayBuffer 视图之一

实例属性

  1. byteLength：返回该数组缓冲区的长度（以字节为单位）

  2. maxByteLength：返回该数组缓冲区可调整到的最大长度（以字节为单位）

  3. resizable：返回此数组缓冲区是否可以调整大小

实例方法

  1. resize()：将 ArrayBuffer 调整为指定的大小，以字节为单位

  2. slice()：返回一个新的 ArrayBuffer，包含原 ArrayBuffer 实例中从 begin（包含）到 end（不含）的所有字节的副本

视图

TypedArray

  1. Int8Array：表示 8 位带符号整数的数组，每个元素占用 1 字节，范围从 -128 到 127。

  2. Uint8Array：表示 8 位无符号整数的数组，每个元素占用 1 字节，范围从 0 到 255。

  3. Uint8ClampedArray：表示 8 位无符号整数的数组，每个元素占用 1 字节，范围从 0 到 255，超出范围的值会被裁剪。

  4. Int16Array：表示 16 位带符号整数的数组，每个元素占用 2 字节，范围从 -32768 到 32767。

  5. Uint16Array：表示 16 位无符号整数的数组，每个元素占用 2 字节，范围从 0 到 65535。

  6. Int32Array：表示 32 位带符号整数的数组，每个元素占用 4 字节，范围从 -2,147,483,648 到 2,147,483,647。

  7. Uint32Array：表示 32 位无符号整数的数组，每个元素占用 4 字节，范围从 0 到 4,294,967,295。

  8. Float32Array：表示 32 位浮点数的数组，每个元素占用 4 字节，精度约为 7 位十进制数字。

  9. Float64Array：表示 64 位浮点数的数组，每个元素占用 8 字节，精度约为 15 位十进制数字。

  10. BigInt64Array：表示 64 位带符号大整数的数组，每个元素占用 8 字节，范围从 $ -2^{63} $ 到 $ 2^{63} - 1 $​​。

  11. BigUint64Array：表示 64 位无符号大整数的数组，每个元素占用 8 字节，范围从 0 到 $ 2^{64} - 1 $​。

DataView

构造函数

  1. buffer： 要创建视图的 ArrayBuffer 对象。这个缓冲区是 DataView 将用于读取和写入数据的基础
  2. byteOffset（可选）：从 ArrayBuffer 的哪个字节偏移量开始创建 DataView。默认值是 0
  3. byteLength（可选）：字节数组中的元素数。如果未指定，则视图的长度将与缓冲区的长度匹配。

实例方法

  1. get/setInt8：读取/写入指定偏移量的 8 位带符号整数
  2. get/setUint8：读取/写入指定偏移量的 8 位无符号整数
  3. get/setInt16：读取/写入指定偏移量的 16 位带符号整数
  4. get/setUint16：读取/写入指定偏移量的 16 位无符号整数
  5. get/setInt32：读取/写入指定偏移量的 32 位带符号整数
  6. get/setUint32：读取/写入指定偏移量的 32 位无符号整数
  7. get/setFloat16：读取/写入指定偏移量的 16 位浮点数
  8. get/setFloat32：读取/写入指定偏移量的 32 位浮点数
  9. get/setFloat64：读取/写入指定偏移量的 64 位浮点数
  10. get/setBigInt64：读取/写入指定偏移量的 64 位带符号大整数
  11. get/setBigUint64：读取/写入指定偏移量的 64 位无符号大整数

注意：

  1. get()有两个参数：第一个参数表示偏移量，第二个参数表示字节序；set()有三个参数：第一个参数表示偏移量，第二个参数表示要设置的值，第三个参数表示数据以什么字节序格式来存储

  2. Int8和Uint8没有字节序，因此get()只有一个参数，set()有两个参数

  3. 字节序指的是在存储多字节数据时，字节的顺序排列方式。通常用布尔值表示，true表示小端序，false表示大端序

SharedArrayBuffer

SharedArrayBuffer表示通用的原始二进制数据缓冲区，类似于 ArrayBuffer 对象，需要通过TypedArray或DataView对象来操作。、

构造函数

  1. length：要创建的数组缓冲区的大小（以字节为单位）
  2. options（可选）：一个对象，可以包含maxByteLength属性，表示数据缓冲区可以调整的最大大小，以字节为单位

实例属性

  1. byteLength：返回该数组缓冲区的长度（以字节为单位）

  2. maxByteLength：返回该数组缓冲区可调整到的最大长度（以字节为单位）

实例方法

  1. slice()：返回一个新的 SharedArrayBuffer，包含原 ArrayBuffer 实例中从 begin（包含）到 end（不含）的所有字节的副本

ArrayBuffer与SharedArrayBuffer区别

  1. 创建和大小
    • ArrayBuffer：通过 new ArrayBuffer(size) 创建，其中size是缓冲区的字节大小，一旦创建，ArrayBuffer 的大小不可更改
    • SharedArrayBuffer：通过 new SharedArrayBuffer(size) 创建，其中 size 是缓冲区的字节大小，SharedArrayBuffer 的大小在创建时固定，无法动态调整
  2. 并发和内存共享
    • ArrayBuffer：单线程环境中使用的固定大小的内存缓冲区。它不支持直接在多个线程之间共享数据
    • SharedArrayBuffer：可以在多个线程之间共享内存。它允许不同线程访问相同的内存区域
  3. 安全和兼容性
    • ArrayBuffer：在所有现代浏览器中都被广泛支持，不受特别的安全限制
    • SharedArrayBuffer：SharedArrayBuffer 在某些浏览器中可能会由于安全性问题被禁用。需要设置特定的 HTTP 头部（Cross-Origin-Opener-Policy 和 Cross-Origin-Embedder-Policy）才能启用。浏览器可能需要在一些环境中进行额外的配置。

Atomics

Atomics 提供了一组原子操作的方法，用于在多线程环境中操作共享内存。这些操作可以在 SharedArrayBuffer 上执行，以确保数据的一致性和正确性。

  1. Atomics.load()：从 typedArray 的指定 index 位置读取值

  2. Atomics.store(typedArray, index, value)：将 value 存储到 typedArray 的指定 index 位置

  3. Atomics.add(typedArray, index, value)：将 value 加到 typedArray 的指定 index 位置的当前值，并返回旧值

  4. Atomics.sub(typedArray, index, value)：从 typedArray 的指定 index 位置的当前值中减去 value，并返回旧值

  5. Atomics.and(typedArray, index, value)：对 typedArray 的指定 index 位置的当前值进行位与操作（&），并返回旧值

  6. Atomics.or(typedArray, index, value)：对 typedArray 的指定 index 位置的当前值进行位或操作（|），并返回旧值

  7. Atomics.xor(typedArray, index, value)：对 typedArray 的指定 index 位置的当前值进行位异或操作（^），并返回旧值

  8. Atomics.exchange(typedArray, index, value)：将 typedArray 的指定 index 位置的值替换为 value，并返回旧值

  9. Atomics.compareExchange(typedArray, index, expectedValue, newValue)：将 typedArray 的指定 index 位置的值替换为 newValue，当且仅当当前位置的值等于 expectedValue。返回当前值

  10. Atomics.wait(typedArray, index, value, timeout)：阻塞当前线程，直到 typedArray 的指定 index 位置的值变为 value，或者超时（如果指定了 timeout）

  11. Atomics.waitAsync(typedArray, index, value, timeout)：异步等待，直到 typedArray 的指定 index 位置的值变为 value，或者超时（如果指定了 timeout）

  12. Atomics.isLockFree(size)：检测当前平台是否能够在不使用锁的情况下执行size大小的原子操作

  13. Atomics.notify(typedArray, index, count)：唤醒在共享内存中的某个位置上等待的线程。这个方法与 Atomics.wait() 配合使用，用于在多线程环境中管理线程间的同步

JSON

JSON 是一种语法，用来序列化对象、数组、数值、字符串、布尔值和 null。基于 JavaScript 语法，但与之不同

基本语法

  1. 数据以键值对形式出现
    • 键（key）是字符串
    • 值（value）可以是字符串、数字、布尔值、null、对象（JSON 对象）、数组（JSON 数组）
  2. 键值之间以冒号分隔
  3. 每个键值对之间用逗号分隔
  4. JSON 数据使用大括号 {} 表示对象，使用方括号 [] 表示数组

静态方法

  1. JSON.stringify()：将 JavaScript 对象或数组转换为 JSON 字符串

  2. JSON.parse()：将 JSON 字符串转换为 JavaScript 对象或数组。
