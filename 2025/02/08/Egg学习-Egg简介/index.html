Egg学习-Egg简介

Egg.js是阿里旗下为企业级框架和应用而生的开源Node.js框架。Egg（简写）奉行【约定优于配置】，Egg并不是集成式框架，而是专注于提供 Web 开发的核心功能和一套灵活可扩展的插件机制，插件机制使Egg具有很高的可扩展性。

Egg简介

特性

  1. 约定优于配置：通过目录结构和命名约定，减少配置文件的复杂性。
  2. 插件机制：通过插件扩展框架功能，保持核心简洁。
  3. 多进程管理：内置多进程模型，支持多核CPU的充分利用。
  4. 渐进式开发：支持从简单到复杂的渐进式开发模式。

Egg.js核心概念

Router

路由定义了 请求路径（URL） 和 控制器（Controller） 之间的映射关系，即用户访问的网址应交由哪个控制器进行处理。

Controller

Controller 负责解析用户的输入，处理后返回相应的结果

Service

Service 是具体业务逻辑的实现，一个封装好的 Service 可供多个 Controller 调用，而一个 Controller 里面也可以调用多个 Service。

Plugin

Egg.js 通过插件机制扩展功能，插件可以是一个NPM包，也可以是一个本地模块。插件可以扩展框架的核心功能，如数据库连接、缓存、日志等。

Config

Egg.js 允许在 config 目录下配置不同的环境变量。
