Egg学习-Router

在 Egg.js 框架中，路由（Router）是请求分发的核心组件，它负责将客户端的 HTTP 请求映射到对应的控制器（Controller）方法上。

Router

在 Egg.js 中，所有的路由配置都集中在 app/router.js 文件中。框架基于 Koa 的中间件机制对路由进行封装，使得路由配置变得简单而灵活。通过路由可以实现：

  1. 定义 URL 与控制器方法的映射
  2. 支持各种 HTTP 方法（GET、POST、PUT、DELETE 等）
  3. 提取 URL 中的动态参数
  4. 在路由中添加中间件，实现请求预处理

基础路由配置

路由定义方式

Egg.js 的路由统一在 app/router.js 中配置，通过 app.router 对象的方法定义。

Router 详细定义说明

  1. verb - 用户触发动作，支持所有HTPP方法
    • router.head - HEAD
    • router.options - OPTIONS
    • router.get - GET
    • router.put - PUT
    • router.post - POST
    • router.patch - PATCH
    • router.delete - DELETE
    • router.redirect 对URL进行重定向处理
  2. router-name（可选）路由别名
  3. path-match 路由URL路径
  4. middleware（可选）可以配置多个middleware
  5. controller 指定路由映射到具体的controller上
