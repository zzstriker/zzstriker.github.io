Egg学习-配置

Egg.js 提供了一套灵活且强大的配置机制，支持多环境管理、插件配置、中间件定义、自定义扩展等场景。

配置

概述

Egg.js 的配置采用 集中化管理，所有的配置都存放在 config/ 目录下，主要包括：

  1. config.default.js：应用的默认配置，适用于所有环境。
  2. config.[env].js：不同环境的特殊配置，如 config.prod.js（生产环境）、config.local.js（本地开发环境）。
  3. config.unittest.js：单元测试环境的配置。
  4. config.secret.js（可选）：存放敏感信息，避免将 API Key、数据库密码等硬编码到代码中。
  5. config.plugin.js：插件相关的配置。

Egg.js 会根据 process.env.EGG_SERVER_ENV 变量自动加载对应的环境配置。

配置文件结构

Egg.js 的配置统一存放在项目的 config 目录下，框架会根据运行环境自动加载对应的配置文件。

默认配置

Egg.js 的默认配置适用于所有环境

多环境配置管理

Egg.js 通过环境变量 EGG_SERVER_ENV 区分不同环境，常见的环境包括：

  1. local：本地开发环境（默认）。
  2. unittest：测试环境。
  3. prod：生产环境。

配置合并规则

  1. 加载顺序：config.default.js → config.{env}.js → config.local.js。
  2. 同名覆盖：后加载的配置会覆盖前面的同名属性。
  3. 深层合并：对象类型的配置会递归合并（而非直接覆盖）。
